This guide covers deploying both the NestJS API and React dashboard to production.

## Pre-deployment Checklist

<Steps>

<Step title="Environment variables">

Ensure you have production values for:

```bash
# Database
DATABASE_URL=postgresql://user:password@host:5432/dbname

# Better Auth
BETTER_AUTH_SECRET=your-production-secret
BETTER_AUTH_URL=https://api.yourapp.com

# Application
NODE_ENV=production
PORT=8000

# OAuth (if using)
GOOGLE_CLIENT_ID=your-production-google-id
GOOGLE_CLIENT_SECRET=your-production-google-secret

# Frontend URL (for CORS)
FRONTEND_URL=https://app.yourapp.com
```

</Step>

<Step title="Build the projects">

Test builds locally:

```bash
# Build all projects
pnpm build

# Or build individually
cd apps/api && pnpm build
cd apps/dashboard && pnpm build
```

</Step>

<Step title="Run migrations">

Apply database migrations:

```bash
cd apps/api
pnpm db:migrate
```

</Step>

</Steps>

## Deployment Options

### Option 1: Railway (Recommended)

Railway is great for full-stack monorepos.

<Steps>

<Step title="Install Railway CLI">

```bash
npm i -g @railway/cli
railway login
```

</Step>

<Step title="Create new project">

```bash
railway init
```

</Step>

<Step title="Add PostgreSQL">

In Railway dashboard:
1. Click "New"
2. Select "Database" → "PostgreSQL"
3. Copy the connection string

</Step>

<Step title="Configure API service">

Create `railway.json` in `apps/api/`:

```json
{
  "build": {
    "builder": "NIXPACKS",
    "buildCommand": "pnpm install && pnpm build"
  },
  "deploy": {
    "startCommand": "pnpm start:prod",
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 10
  }
}
```

</Step>

<Step title="Configure Dashboard service">

Create `railway.json` in `apps/dashboard/`:

```json
{
  "build": {
    "builder": "NIXPACKS",
    "buildCommand": "pnpm install && pnpm build"
  },
  "deploy": {
    "startCommand": "pnpm preview",
    "restartPolicyType": "ON_FAILURE"
  }
}
```

</Step>

<Step title="Set environment variables">

In Railway dashboard, add all environment variables from the checklist.

</Step>

<Step title="Deploy">

```bash
railway up
```

</Step>

</Steps>

### Option 2: Docker

Deploy using Docker containers.

<Steps>

<Step title="Create API Dockerfile">

Create `apps/api/Dockerfile`:

```dockerfile
FROM node:20-alpine AS builder

WORKDIR /app

# Install pnpm
RUN corepack enable && corepack prepare pnpm@latest --activate

# Copy package files
COPY package.json pnpm-lock.yaml pnpm-workspace.yaml ./
COPY apps/api/package.json ./apps/api/

# Install dependencies
RUN pnpm install --frozen-lockfile

# Copy source
COPY apps/api ./apps/api
COPY packages ./packages

# Build
RUN pnpm --filter @repo/api build

# Production stage
FROM node:20-alpine

WORKDIR /app

RUN corepack enable && corepack prepare pnpm@latest --activate

# Copy built files
COPY --from=builder /app/apps/api/dist ./dist
COPY --from=builder /app/apps/api/package.json ./
COPY --from=builder /app/node_modules ./node_modules

EXPOSE 8000

CMD ["node", "dist/main"]
```

</Step>

<Step title="Create Dashboard Dockerfile">

Create `apps/dashboard/Dockerfile`:

```dockerfile
FROM node:20-alpine AS builder

WORKDIR /app

RUN corepack enable && corepack prepare pnpm@latest --activate

COPY package.json pnpm-lock.yaml pnpm-workspace.yaml ./
COPY apps/dashboard/package.json ./apps/dashboard/

RUN pnpm install --frozen-lockfile

COPY apps/dashboard ./apps/dashboard
COPY packages ./packages

RUN pnpm --filter @repo/dashboard build

# Production stage
FROM nginx:alpine

COPY --from=builder /app/apps/dashboard/dist /usr/share/nginx/html
COPY apps/dashboard/nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

</Step>

<Step title="Create nginx config">

Create `apps/dashboard/nginx.conf`:

```nginx
server {
    listen 80;
    server_name _;
    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://api:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

</Step>

<Step title="Build and run">

```bash
# Build images
docker build -t sunny-api ./apps/api
docker build -t sunny-dashboard ./apps/dashboard

# Run with docker-compose
docker-compose -f docker-compose.prod.yml up -d
```

</Step>

</Steps>

### Option 3: Vercel + Render

Deploy frontend to Vercel, backend to Render.

<Steps>

<Step title="Deploy Dashboard to Vercel">

```bash
cd apps/dashboard
pnpm i -g vercel
vercel
```

Configure:
- **Build Command**: `pnpm build`
- **Output Directory**: `dist`
- **Install Command**: `pnpm install`

</Step>

<Step title="Deploy API to Render">

1. Connect your GitHub repo to Render
2. Create new "Web Service"
3. Configure:
   - **Root Directory**: `apps/api`
   - **Build Command**: `pnpm install && pnpm build`
   - **Start Command**: `pnpm start:prod`
   - **Environment**: Node 20
4. Add environment variables
5. Add PostgreSQL database

</Step>

<Step title="Update CORS">

Update your frontend URL in the API's CORS config to allow your Vercel domain.

</Step>

</Steps>

## Build Commands

From root of monorepo (package.json:5-11):

```bash
# Build everything
pnpm build

# Build API only
pnpm --filter @repo/api build

# Build Dashboard only
pnpm --filter @repo/dashboard build

# Type checking
pnpm typecheck

# Linting
pnpm lint
```

## Production Start Commands

### API (api/package.json:14)

```bash
cd apps/api
pnpm start:prod
```

This runs the compiled JavaScript from the `dist/` folder.

### Dashboard (dashboard/package.json:8)

```bash
cd apps/dashboard
pnpm preview
```

This serves the built static files.

## Database Migrations in Production

<Warning>
  Always backup your database before running migrations in production.
</Warning>

```bash
# From apps/api directory
DATABASE_URL=your-prod-url pnpm db:migrate
```

For zero-downtime deployments:
1. Deploy new code
2. Run migrations
3. Restart services

```bash
NODE_ENV=production
PORT=8000

BETTER_AUTH_SECRET=${BETTER_AUTH_SECRET}
BETTER_AUTH_URL=https://api.yourapp.com
```

Load in production:

```bash
NODE_ENV=production node dist/main
```

## Health Checks

Add a health check endpoint to your API:

```typescript
// apps/api/src/app.controller.ts
import { Controller, Get } from '@nestjs/common';

@Controller()
export class AppController {
  @Get('health')
  health() {
    return { status: 'ok', timestamp: new Date().toISOString() };
  }
}
```

Use for:
- Railway health checks
- Docker health checks
- Monitoring services

## Monitoring

Add error tracking:

```bash
pnpm add @sentry/node
```

In `main.ts`:

```typescript
import * as Sentry from '@sentry/node';

if (process.env.NODE_ENV === 'production') {
  Sentry.init({
    dsn: process.env.SENTRY_DSN,
    environment: 'production',
  });
}
```

## Common Issues

### Build Fails

Check:
- All dependencies are in `package.json`
- TypeScript compiles with `pnpm typecheck`
- Environment variables are set

### Database Connection Fails

Verify:
- `DATABASE_URL` is correct
- Database accepts connections from your host
- SSL settings match your provider

### CORS Errors

Update CORS config to include production URLs:

```typescript
// apps/api/src/config/cors.config.ts
export const trustedOrigins = [
  'http://localhost:3000',
  'https://app.yourapp.com', // Add production URL
];
```
